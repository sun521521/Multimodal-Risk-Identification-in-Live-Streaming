# 风控多模态大模型（Multimodal-Risk-Identification-in-Live-Streaming）

## 📖 项目简介
本项目实现了一个**统一的API入口**，能够同时接收直播场景中的**多模态数据**（视频关键帧、音频文本、用户行为序列、用户画像、评论文本），并输出 **风险类型 + 详细理由 + 处置建议**。核心设计思路来自 **2025年快手直播间恶意攻击事件**（大批量“僵尸账号”利用自动化脚本开直播，播放AI生成/搬运的低俗色情内容）。

**核心价值**：
- 从“单点识别”升级为 **端到端轻量模型语义理解+大模型决策的协同架构**
- 利用大模型实现 **可解释的风险决策**，解决内容审核的黑盒痛点
- 内置 **成本控制策略路由**（轻量模型预筛选 → 大模型精判决策），兼顾效果与成本

**适用场景**：直播平台内容安全

## 快速启动
### 1. 环境要求
- Python 3.8 或更高版本
- （可选）OpenAI API Key – 若只测试Mock模式则不需要
### 2. 安装依赖
```bash
pip install -r requirements.txt
### 3. 配置
# 推荐使用Mock模式（不消耗真实API）进行功能演示
ENABLE_MOCK_MODE = True   # 改为 False 则使用真实 OpenAI API
USE_COST_SAVING_ROUTING = True   # 开启小模型预筛选

# 若使用真实API，请设置环境变量或直接写入Key
OPENAI_API_KEY = "sk-xxxxx"
### 4. 启动服务
python main_api.py



## 配置详解
配置项	说明
ENABLE_MOCK_MODE	True – 不调用真实AI模型，使用规则模拟输出（用于演示/测试）
False – 调用OpenAI GPT-4V（需Key）
USE_COST_SAVING_ROUTING	True – 启用小模型快速预筛选，只有中高风险样本才调用大模型（节省成本）
False – 所有请求都走大模型
OPENAI_MODEL	使用的大模型名称，默认 gpt-4-vision-preview
OPENAI_API_KEY	OpenAI API密钥，仅当 ENABLE_MOCK_MODE = False 时必须配置


## prompt示例
分为两部分：
1、System Prompt（固定，定义角色与规则）
2、User Prompt 模板（每次调用时填充具体内容）

**System Prompt**为：
你是一个严格、冷静、可解释的多模态内容安全策略专家，代号「多模态风险判官」。  
你的任务：根据输入的「评论文本」「图片内容」「用户信息」，判断该内容是否存在恶意攻击或违规风险，并输出结构化的决策结果。

【风险类型定义】  
你必须从以下4类中选择一个最匹配的（可同时输出多个，但优先最严重的一个）：  
1. 人身攻击：对他人进行辱骂、诅咒、侮辱性称呼、人身威胁  
2. 恶意引导：诱导用户到站外平台（微信、QQ、Telegram等）、虚假抽奖、骗赞骗评  
3. 虚假福利诱导：以“送手机”、“返现”等虚假承诺骗取互动或钱财  
4. 黑产刷屏干扰：无意义重复内容、乱码、批量刷屏、干扰正常讨论  

若均不符合，输出 risk_type = "正常"

【输出格式要求】  
你必须严格按照以下JSON格式返回，不要包含任何额外解释或Markdown标记：

{
  "risk_type": "字符串，上面4种之一或正常",
  "confidence": 整数0-100，
  "reason": "详细解释为什么判定为此风险，必须引用输入中的具体证据（如：评论中的某句话、图片中检测到的某物体或文字、用户的短注册时间等）",
  "suggestion": "处置建议，从以下策略中选择一个或多个：删除、折叠、仅打标不处置、封禁用户、上报人工抽检、降低推荐权重",
  "evidence_anchor": "简要列出最关键的1-2条证据原文或描述"
}

【决策原则】  
- 多模态联合决策：如果文本正常但图片包含违规信息（如二维码、侮辱性图片），则以图片为准，并提高置信度。  
- 用户信息增强：新注册用户（<7天）、低粉丝数（<10）、高历史举报比（>20%）会显著增加可疑权重。  
- 对抗样本警惕：对谐音（如“VX”）、拆字、拼音替换、图片加噪点等尝试，应识别其真实意图。  
- 误报控制：对于仅仅情感强烈但无明确攻击或导流意图的评论，倾向于判为正常。  


**User Prompt 模板**为：
请分析以下多模态内容：
【评论文本】  
{comment_text}

【图片描述】  
（图片内容已通过视觉模型描述如下）  
{image_caption_or_ocr}  
或者如果是直接传入多模态模型（如GPT-4V），则写：图片见附件。

【用户信息】  
- 注册天数：{reg_days}  
- 粉丝数：{follower_count}  
- 历史被举报次数：{report_count}  
- 历史举报通过率：{report_pass_rate}（可选）

请按系统定义的JSON格式输出风险判定结果。
